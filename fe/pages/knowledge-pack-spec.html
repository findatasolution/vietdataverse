Chuẩn cấu trúc Knowledge Pack

Hướng dẫn kỹ thuật cho Seller — định nghĩa cấu trúc, yêu cầu nội dung và quy trình kiểm tra tự động áp dụng khi upload sản phẩm lên Viet Dataverse.

Tổng quan: Knowledge Pack là gì?

Knowledge Pack là file chứa kiến thức chuyên biệt về thị trường tài chính Việt Nam, được đóng gói theo chuẩn để AI agent có thể load và sử dụng ngay mà không cần tiền xử lý thêm.

Khác với prompt thông thường, một Knowledge Pack có đầy đủ: cấu trúc rõ ràng, code snippet sẵn chạy được, prompt snippet định nghĩa hành vi agent, và tham chiếu nguồn dữ liệu thực tế từ Viet Dataverse API.

Tương thích với: Claude, GPT-4o, Cursor, n8n, Dify, LangChain, CrewAI — bất kỳ LLM pipeline nào hỗ trợ system context injection.

Tại sao cần chuẩn cấu trúc?

System kiểm tra tự động (auto-validator) chạy ngay khi Seller upload. Pack đạt chuẩn thì lên sàn ngay, không cần admin duyệt. Pack thiếu phần bắt buộc bị reject với thông báo chi tiết để sửa.

3 loại pack

Viet Dataverse hỗ trợ ba định dạng sản phẩm, mỗi loại phục vụ một use case khác nhau trong AI agent pipeline:

Knowledge Document

Tài liệu kiến thức

Chứa facts, data ranges, công thức tính, cơ chế thị trường VN. Agent load vào system context để reasoning. Phù hợp cho bất kỳ LLM nào.

Định dạng: .md
Agent Blueprint

Bản thiết kế Agent

Định nghĩa toàn bộ hành vi agent — tools, routing logic, system prompt, few-shot examples. Import trực tiếp vào LangChain, n8n, Dify như workflow definition.

Định dạng: .yaml / .json
Dataset Pack

Bộ dữ liệu lịch sử

Bảng dữ liệu timeseries — lãi suất theo năm, CPI series, VN30 benchmark. Agent dùng cho RAG lookup, few-shot context, hoặc backtest pipeline.

Định dạng: .csv / .json

Loại 1: Knowledge Document (.md)

Knowledge Document là loại pack phổ biến nhất. File Markdown chứa kiến thức chuyên sâu về một chủ đề tài chính VN, cấu trúc theo 7 phần bắt buộc để developer và researcher đều dùng được ngay.

Cấu trúc skeleton

# [Tên chủ đề]
**Dành cho:** [use case chính — 1 câu]
**Tầm quan trọng:** [tại sao quan trọng với nền kinh tế VN — 1-2 câu]

## Cách dùng pack này

### Dành cho Developer / Agent Builder
```python
import requests

API_KEY = "your-api-key"
BASE    = "https://api.vietdataverse.online/api/v1"

# Lấy 90 ngày giá vàng SJC
r = requests.get(f"{BASE}/gold",
    params={"type": "SJC", "period": "90d"},
    headers={"X-API-Key": API_KEY})

latest = data[-1]
print(f"SJC {latest['period']}: mua={latest['buy_price']:,} | bán={latest['sell_price']:,} VND/lượng")
# Output: SJC 2025-06-08: mua=121,500,000 | bán=123,000,000 VND/lượng

BTMC         24K 1 lượng 119,700,000  121,200,000   1,500,000 VND

Chênh lệch nội địa–quốc tế: ~10–13 triệu VND/lượng (khoảng 10–12%)

Nguồn: Viet Dataverse API / SJC / DOJI / BTMC, tháng 6/2025
```

Loại 2: Agent Blueprint (.yaml / .json)

Agent Blueprint là file YAML hoặc JSON định nghĩa toàn bộ hành vi của một AI agent chuyên biệt. Khác với Knowledge Document chỉ là reference material, Blueprint có thể import trực tiếp vào LangChain, n8n, Dify như một workflow definition — tools, routing logic, few-shot examples đều được khai báo sẵn.

Import được vào: LangChain (agent config), n8n (workflow JSON), Dify (agent YAML), Claude Tools, bất kỳ pipeline nào hỗ trợ YAML/JSON agent definition.

Cấu trúc skeleton

# VDV Agent Blueprint v1
meta:
  name: "Tên agent"
  version: "1.0"
  description: "Mô tả 1 câu — agent làm gì, cho ai"
  domains: [macro, gold, fx]        # lĩnh vực chuyên môn
  compatible_with: [langchain, n8n, dify, claude-tools]

system_prompt: |
  Bạn là agent phân tích [lĩnh vực] cho thị trường Việt Nam.
  Chuyên môn: [liệt kê cụ thể]
  Nguyên tắc trả lời:
  - Luôn trích dẫn nguồn dữ liệu
  - Chỉ cung cấp thông tin tham khảo
  KHÔNG đưa ra khuyến nghị mua/bán.

tools:
  - name: get_gold_price
    description: "Lấy giá vàng VN từ Viet Dataverse"
    endpoint: "https://api.vietdataverse.online/api/v1/gold"
    params: {type: string, period: string}
    auth: x-api-key

  - name: get_fx_rate
    description: "Lấy tỷ giá USD/VND"
    endpoint: "https://api.vietdataverse.online/api/v1/sbv-centralrate"
    params: {currency: string, period: string}
    auth: x-api-key

routing:
  - intent: "hỏi về vàng"
    tool: get_gold_price
    then: analyze_and_respond
  - intent: "hỏi về tỷ giá"
    tool: get_fx_rate
    then: analyze_and_respond

few_shot_examples:
  - user: "Giá vàng SJC hôm nay bao nhiêu?"
    assistant: |
      Dựa trên dữ liệu từ Viet Dataverse (cập nhật [ngày]):
      - SJC mua vào: 120,500,000 VND/lượng
      - SJC bán ra: 122,500,000 VND/lượng
      Chênh lệch so với giá quốc tế: khoảng 11 triệu VND/lượng (~10%).

disclaimer: |
  Agent này chỉ cung cấp thông tin tham khảo.
  Không chịu trách nhiệm về các quyết định đầu tư dựa trên thông tin từ agent.

Ví dụ thực tế — Credit Scoring Support Agent (rút gọn)

# VDV Agent Blueprint v1
meta:
  name: "VN Credit Context Agent"
  version: "1.2"
  description: "Agent cung cấp context lãi suất & tín dụng VN cho hệ thống credit scoring"
  domains: [finance, credit, termdepo, sbv-policy]
  compatible_with: [langchain, n8n, dify, openai-function-calling]

system_prompt: |
  Bạn là agent chuyên cung cấp dữ liệu nền tảng về thị trường tín dụng Việt Nam.
  Mục tiêu: hỗ trợ mô hình credit scoring bằng cách cung cấp context lãi suất thực tế.

  Khi được hỏi, luôn:
  1. Truy vấn dữ liệu mới nhất từ Viet Dataverse
  2. Cung cấp khoảng lãi suất thị trường theo kỳ hạn
  3. Nêu xu hướng 3 tháng gần nhất
  4. Đặt trong bối cảnh chính sách SBV hiện hành

  KHÔNG đưa ra đánh giá tín dụng cá nhân.
  KHÔNG đưa ra khuyến nghị cho vay hay từ chối vay.

tools:
  - name: get_termdepo_rates
    description: "Lấy lãi suất tiết kiệm ngân hàng theo kỳ hạn"
    endpoint: "https://api.vietdataverse.online/api/v1/termdepo"
    params:
      period: {type: string, options: [30d, 90d, 1y], default: 90d}
    auth: x-api-key
    output_format: |
      Trả về mảng [{period, term_1m, term_3m, term_6m, term_12m}]

  - name: get_sbv_policy_rate
    description: "Lấy lãi suất điều hành SBV"
    endpoint: "https://api.vietdataverse.online/api/v1/sbv-interbank"
    params:
      period: {type: string, default: 90d}
    auth: x-api-key

routing:
  - intent: "lãi suất hiện tại"
    tool: get_termdepo_rates
    then: summarize_rates
  - intent: "lãi suất SBV"
    tool: get_sbv_policy_rate
    then: summarize_rates
  - intent: "so sánh kỳ hạn"
    tool: get_termdepo_rates
    then: compare_tenors

few_shot_examples:
  - user: "Lãi suất tiết kiệm 12 tháng hiện nay là bao nhiêu?"
    assistant: |
      Theo dữ liệu từ Viet Dataverse (ACB, cập nhật tháng 6/2025):
      - 1 tháng: 3.1%/năm
      - 3 tháng: 3.5%/năm
      - 6 tháng: 4.2%/năm
      - 12 tháng: 5.0%/năm
      Xu hướng 3 tháng: lãi suất 12 tháng tăng nhẹ +0.3% so với tháng 3/2025.

disclaimer: |
  Dữ liệu lấy từ Viet Dataverse API, có thể trễ 1 ngày so với niêm yết thực tế.
  Chỉ dùng cho mục đích tham khảo trong credit scoring pipeline.
  Không thay thế thẩm định tín dụng chuyên nghiệp.

Cấu trúc bắt buộc (7 phần)

Áp dụng cho Knowledge Document (.md). Auto-validator kiểm tra tất cả 7 phần theo thứ tự khi upload.

System tự động reject pack thiếu phần bắt buộc. Bạn sẽ nhận thông báo chi tiết check nào fail và cách khắc phục — không cần liên hệ admin.
# Phần Bắt buộc? Mô tả Áp dụng cho

**Dành cho:**
**Tầm quan trọng:**		 Bắt buộc 3 dòng header. **Dành cho:** phải xuất hiện trong 5 dòng đầu file. Knowledge Document

Checklist trước khi submit

Tự kiểm tra trước khi upload để tránh bị reject:

  • Header có **Dành cho:** và **Tầm quan trọng:** trong 5 dòng đầu
  • Mục ## Cách dùng pack này có đủ 2 sub-section (Developer + Researcher)
  • Code Python trong phần Developer có ít nhất 10 dòng và chạy được
  • Có ít nhất 3 câu hỏi gợi ý cụ thể (không hỏi chung chung)
  • Có ít nhất 2 phần nội dung chính với heading ##
  • Mọi số liệu đều kèm tên nguồn và năm
  • Có mục ## Prompt snippet cho agent với tối thiểu 10 dòng
  • Prompt snippet kết thúc bằng dòng disclaimer "KHÔNG đưa ra khuyến nghị..."
  • Có mục ## Nguồn dữ liệu với ít nhất 1 URL hoặc endpoint thực tế
  • File size từ 2 KB đến 50 KB
  • Không chứa API key thật, SĐT cá nhân, CMND, hoặc thông tin bí mật khác
  • Mô tả sản phẩm (field "Mô tả" trên form upload) có ít nhất 50 ký tự

Cách upload

Quy trình upload 3 bước — không có admin review, pack lên sàn ngay khi pass validator:

1
Upload file — Vào Seller Dashboard, click "Upload sản phẩm mới", điền form (tiêu đề, mô tả, giá, category) và chọn file. Click "Upload".
2
Auto validate — System chạy 4 bước kiểm tra trong vài giây: kiểm tra mã độc, parse định dạng, kiểm tra cấu trúc 7 phần, quét PII và secrets. Nếu fail, modal hiển thị check nào fail và hướng dẫn sửa.
3
Live ngay — Pass tất cả checks, sản phẩm lên sàn ngay với status "Published". Không cần chờ admin duyệt. Buyer có thể xem preview và mua trong vài phút.
Xem các pack mẫu VD Official (packs 01–13) trong Knowledge Market để tham khảo ví dụ thực tế đã được publish và đạt chuẩn validator.
